  • https://lalith.in/2020/09/27/Low-Level-Advice-For-Systems-Research

    • prepare your experiment infrastructure as early as possible
    • goal of prototypes is to exercise a certain hypothesis
    • automate experiment workflow
    • as systems get complex, ensure correctness with tests
    • avoid tunnel vision.
      • write first few paragraphs of paper's evaluation section
      • Compare results of experiments from the system's same version
    • use feature flags
    • check features individually
    •  John Ousterhout: “Use your intuition to ask questions, not answer them”. Log extensively.
    • Recommended reads:
      • “Performance Evaluation of Computer and Communication Systems”, Jean-Yves Le Boudec.
      • “The Art of Computer Systems Performance Analysis: Techniques for Experimental Design, Measurement, Simulation, and Modeling”, Raj Jain.
      • “Mathematical foundations of computer networking”, Srinivasan Keshav.
      • “Systems Benchmarking Crimes”, Gernot Heiser

  •  “Systems Benchmarking Crimes”, Gernot Heiser

    • performance should improve significantly in the area of interest
    • performance should not significantly degrade elsewhere
    • demonstrate acceptability of new overheads
    • pick experiments justly
    • don't downplay/sugarcoat overheads
    • show other stats, use geometric means
    • split train and test (calibration and eval. it is called)
    • consider baseline performance carefully
    • when running other app be sure it doesn't have debugging and stuff like that enabled. Be sure that current app is the best working version of this baseline artifact
    • document what you are doing

#!/bin/bash
#push_notes_to_web.sh 
OBSIDIAN_EXPORT="/home/matin/Documents/iztech/masters/thesis/web_notes"
REPO_THESIS="/home/matin/Documents/github/matinhuseynzade.com/src/private/thesis"
REPO="/home/matin/Documents/github/matinhuseynzade.com"

# 1. Check if source is empty
if [ -z "$(ls -A "$OBSIDIAN_EXPORT")" ]; then
  echo "Source is empty, nothing to do."
  exit 0
fi

# 2. Sync to repo
rsync -av --delete --exclude='index.html' "$OBSIDIAN_EXPORT/" "$REPO_THESIS/"

# 3. Clean source folder
rm -rf "$OBSIDIAN_EXPORT"/*

# 4. Git push
cd "$REPO"
git add src/private/thesis/
git commit -m "chore: sync thesis $(date '+%Y-%m-%d %H:%M')" || echo "Nothing to commit"
git push
